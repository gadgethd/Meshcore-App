# MeshCore Desktop

Cross-platform desktop app for working with MeshCore companion radios on Linux, macOS, and Windows.

## Current Status

`0.0.3-alpha`

The app is usable for early desktop testing and companion-radio workflows. USB is the most reliable transport today. Bluetooth support exists, but it is still host-stack dependent and remains experimental, especially on Linux.

## Features

- Auto-detect and connect to MeshCore companion radios over USB serial
- Bluetooth transport support in the desktop app
- Direct messages and channel messages
- Unread counts with per-node read tracking
- Packet/message detail dialog with hop count and known route hop codes when available
- Local 14-day message archive for desktop history
- Live diagnostics feed for adverts, messages, sync activity, probes, and connection events
- Dark network map with live and archived nodes
- Companion radio settings editor
- In-app update controls for packaged builds
- Background-running mode with tray behavior and desktop notifications

## Platform Targets

- Linux: `AppImage`
- macOS: `dmg`, `zip`
- Windows: `nsis`

Release builds are produced through GitHub Actions and published to GitHub Releases.

## Development

Requirements:

- Node.js 20+
- npm

Install dependencies:

```bash
npm install
```

Run the app in development:

```bash
npm run dev
```

Type-check:

```bash
npm run typecheck
```

Build production bundles:

```bash
npm run build
```

Package the app:

```bash
npm run package -- --linux AppImage
```

## Project Layout

```text
src/
  main/        Electron main process, updater, serial manager, IPC handlers
  preload/     Typed context bridge exposed as window.meshcoreAPI
  renderer/    React UI, stores, views, and MeshCore desktop UX
  shared/      Shared protocol, message, channel, and app types
```

## Release Notes

For tagged releases:

- tag only after verifying the tag points at the intended `main` commit
- verify `package.json` and `src/shared/app-meta.ts` both contain the intended version
- use curated release notes instead of GitHub autogenerated notes

## Notes

- Linux serial access may require membership in the `dialout` group.
- Linux tray, dock, notifications, and Bluetooth behavior can still vary by desktop environment.
- In-app updates only work in packaged builds and require published updater metadata (`latest*.yml`, `.blockmap`, and macOS zip assets).
- This repository intentionally excludes local machine-specific launcher files and other personal/local workflow state.
